# Risk Register

Risks to the Weather Explorer MVP, with likelihood, impact, mitigation, owner,
and status. Tone is factual: a mitigated risk is still listed so the next
engineer knows it was considered. Likelihood/impact are Low / Medium / High.

Owners: **Eng** = delivery engineer; **Ops** = whoever owns the Vercel
deploy; **Product** = scope owner.

| # | Risk | Likelihood | Impact | Mitigation | Owner | Status |
|---|---|---|---|---|---|---|
| R-01 | **Open-Meteo availability** — if the keyless forecast/geocoding API is down or slow, the app cannot fetch data. | Medium | Medium | Every external call degrades to a calm inline error + retry, never a 500 or blank (`ForecastError.tsx`, `Notice.tsx`, search/compare/map inline states); `fetchForecast`/route handlers never throw. Verified by MT-13 and integration tests. No persistence to fall back on (no-DB by design). | Eng | Mitigated |
| R-02 | **Open-Meteo rate limits / fair-use** — the free tier has a fair-use daily cap; the compare view fetches once per pinned city (up to 3), and map clicks re-fetch. Heavy demo traffic could throttle. | Low | Medium | Request-scoped `cache()` (`getForecast.ts`) dedupes within a render; debounced search (~300 ms) reduces geocode calls; compare is capped at 3 cities. Throttle responses surface as the calm error state. If demo load is high, consider a short server-side cache header at G7. | Eng / Ops | Open (monitor at G7) |
| R-03 | **OSM tile usage policy** — OSM's Tile Usage Policy requires HTTPS, valid attribution, and no bulk scraping; violation risks tile blocking. | Low | Medium | Tiles served over `https://tile.openstreetmap.org` with "© OpenStreetMap contributors" attribution always visible (`MapView.tsx`, TC-MAP-01, FR-MAP-04); city-level zoom and single-marker use, no scraping. | Eng | Mitigated |
| R-04 | **Forecast horizon nulls** — Open-Meteo precipitation probability and UV index are not populated for the full 7-day horizon; later days can carry nulls. | Medium | Low | `lib/weather` mappers tolerate nulls; `DayCard`/`CompareTable` render an em dash for missing values; `comfortScore` falls back to neutral defaults for `?? undefined` inputs and never throws. Covered by `lib/weather/map.test.ts`, `lib/scoring/comfort.test.ts`. | Eng | Mitigated |
| R-05 | **Hydration mismatch** — the live clock and the day-of-year joke could differ between server and client render, producing React hydration warnings (would also break NFR-OBS-01). | Medium | Medium | `Clock` uses `useSyncExternalStore` with a stable server snapshot + `suppressHydrationWarning`; the joke is selected from a stable day-of-year index (no `Math.random`/`Date.now` in `lib`). Smoke confirms a silent console. | Eng | Mitigated |
| R-06 | **Initial JS bundle budget** (NFR-PERF-03 ≤ 200 KB gz) — Leaflet + Recharts are large and could blow the budget if statically imported. | Medium | Medium | Both are loaded via `dynamic({ ssr:false })` (`MapPanel.tsx`, `HourlyChartLazy.tsx`), keeping them out of the initial/empty-state bundle. Build succeeds. **Measure gzipped initial JS on the deployed build at G7.** | Eng / Ops | Open (verify at G7) |
| R-07 | **Lighthouse Performance / Accessibility scores** (NFR-PERF-02 ≥ 90, NFR-A11Y-01 ≥ 95) cannot be measured from the repo; only on the production URL. | Medium | Medium | Code-level mitigations in place (deferred libs, bounded particle count, focus styles, accessible names, AA-contrast tokens). **Run Lighthouse mobile + desktop on the deployed URL at G7**; if a score misses, address before sign-off. | Ops / Eng | Open (G7) |
| R-08 | **Vercel TTFB** (NFR-PERF-01 ≤ 300 ms p95) is deploy-dependent and unmeasurable locally. | Low | Medium | Server Components, no DB, no auth, minimal server work. **Measure p95 homepage TTFB on the Vercel Preview at G7.** | Ops | Open (G7) |
| R-09 | **Reduced-motion compliance** (FR-ANIM-03) — if the particle layer animated despite the OS setting, it would fail accessibility expectations. | Low | Low | Enforced in CSS (`@media (prefers-reduced-motion: reduce)` hides `.sky-particles`), so it holds even before hydration and without JS. Verify by MT-09b. | Eng | Mitigated |
| R-10 | **Map click sets coordinates, not a city name** (FR-MAP-03 as amended) — a viewer may expect a place name on map click. | Low | Low | Accepted and documented (ADR-0004): Open-Meteo has no reverse geocoding; clicked points use a calm rounded-coordinate label. Forecast/comfort/map/background all work from lat/lon. Forward search still yields named cities. | Product / Eng | Accepted (by design) |
| R-11 | **Dev-only advisories in `npm audit`** — transitive dev-dependency advisories (e.g. in the PostCSS/esbuild toolchain) may show in `npm audit`; these affect tooling, not the shipped keyless runtime. | Low | Low | Tracked, not blocking: they are dev-time only and the runtime ships no secrets and no server attack surface beyond the two keyless proxy routes. Re-run `npm audit` at G7 and record the remaining advisories in the delivery report; upgrade if a runtime-affecting advisory appears. | Eng | Open (track at G7) |
| R-12 | **Eval baseline not yet committed** — `check-eval-ratchet` is SKIP; `evals/results/latest.json` exists (generated 2026-06-22) but no `quality/eval-baseline.json`. Baselining now would lock in the current failing case (R-16). | Medium | Low | Fix R-16 first, re-run the `eval-suite` workflow, then commit `quality/eval-baseline.json` so the ratchet guards regressions in CI. | Eng | Open (G6) |
| R-16 | **Comfort rationale wrong on wet days** — `eval-comfort-rationale-wet` FAILS at 42/100: at 95% precipitation `lib/scoring/comfort.ts` returns "Приємний день, комфортно перебувати надворі" (a pleasant day), ignoring the rain (FR-COMFORT-03 accuracy). Unit tests pass because they check format/tone, not accuracy. | High | Medium | Fix the rationale selection so high precipitation produces a rain-aware sentence; add/adjust a unit assertion for the wet branch; re-run the eval suite to confirm the case passes before baselining. | Eng | Open (G6) |
| R-13 | **Demo recordings not yet captured/reviewed** — the matrix cites manual tests + smoke for several FRs; polished recordings are a G6 deliverable. | Medium | Low | Capture per `demo-script.md`, one clip per viewport; **visually review every screenshot and frame-sensitive video moment** before treating a clip as evidence; list FR ids in each manifest entry. | Eng | Open (G6) |
| R-14 | **Geolocation / permissions edge cases** (FR-SEARCH-06) — denial, timeout, or unsupported browser. | Low | Low | Denial/failure falls back silently to the empty state (no toast); button is guarded for missing `navigator.geolocation`. Verify both paths in MT-04. | Eng | Mitigated |
| R-15 | **Stale-response races in search** — a slow earlier geocode response could overwrite a newer one. | Low | Low | Monotonic request-id guard in `CitySearch.tsx` drops all but the latest response. | Eng | Mitigated |
